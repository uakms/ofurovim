ヘブライ語サポート (オプション & マッピング)

’rightleft’ 機能のサポートは初めに Avner Lottem <alottem at gmail dot com> によつて作成されました。現在は Ron Aaron <ron at ronware dot org> がこれらの機能のサポートをしてゐます。

{|+rightleft| 機能を有效にしてコンパイルした場合にのみ利用できます}

はじめに

ヘブライ語に特定のオプションは ’hkmap’, ’hkmapp’, ’keymap’=hebrew と ’aleph’ です。

ヘブライ語に有用なオプションは ’delcombine’, ’allowrevins’, ’revins’, ’rightleft’ と ’rightleftcmd’ です。

’rightleft’ モードは表示する順番を反對にします。そのため、文字は通常、左から右へ表示される代はりに、右から左へ表示されます。これは主にヘブライ語や他の中東地域の言語を編輯するときに有用です。

詳細は |rileft.txt| を參照。

詳細

オプション:

  • ’rightleft’ (’rl’) はウィンドウの方向を右から左に設定します。これは、論理的なテキスト ’ABC’ が ’CBA’ のやうに表示され、ウィンドウの左端からではなく、右端から描畫が開始されることを意味します。
  • ’hkmap’ (’hk’) は插入/置換モードでのヘブライ語向けのキーボドマッピングを設定します。
  • ’aleph’ (’al’) は數値で、キーボードマッピングのための Aleph 10 進數コードを保持します。
  • ’hkmapp’ (’hkp’) は ’表音ヘブライ語’ のキーボードマッピングを設定します。

    Note:
    これら 3 つのオプション (’hkmap’, ’hkmapp’ と ’aleph’) は、もはや使はれません。代はりに ":set keymap=hebrewp" を使用するべきです。

  • ’delcombine’ (’deco’) は切替オプションで、もしも UTF-8 エンコーディングのヘブライ語を編輯してゐるのであれば、(撥音記號を持つ) 文字の上で ’x’ を押すことによつて niqud (撥音記號) や te‘amim (朗唱マーク) を削除できるやうにします。
  • ’rightleftcmd’ (’rlc’) は、檢索時のコマンドプロンプトが右側から現れるやうにします。これは ’rightleft’ であるウィンドウの時にのみ效果があります。

エンコーディング:

  • Unix では ISO 8859-8 エンコーディング (ヘブライ語文字コード: 224-250) です。
  • MS DOS では PC エンコーディング (ヘブライ語文字コード: 128-154) です。これが既定ですが、’aleph’ オプションで變更することができます。
  • 結合文字がサポートされてゐるので、UTF8 を使ひませう (UTF8 エンコーディングが有效でないと、’deco’ が機能しません) 。

Vim への引數:

  • ’vim -H file’ でヘブライ語のファイル編輯を開始します。すなはち、’rightleft’ と ’hkmap’ が設定されます。

キーボード:

  • ’allowrevins’ オプションは、插入モードとコマンドラインモードで CTRL-_ コマンドを有效にします。
  • 插入/置換モードでの CTRL-_ は ’revins’ と ’hkmap’ を次のやうに切り替へます:

    rightleft ウィンドウの時は、’revins’ と ’nohkmap’ がトグルされます。この場合、おそらく英語が入力されるであらうからです。

    rightleft ではないウィンドウの時は、’revins’ と ’hkmap’ がトグルされます。この場合、おそらくヘブライ語が入力されるであらうからです。

    CTRL-_ はカーソルを入力されたテキストの最後に移動します。

  • コマンドモード中の CTRL-_ はキーボードマッピングのみをトグルします (下記の Bugs を參照)。この設定は ’hkmap’ オプションとは獨立してゐて、これは插入/置換モードでのみ適用します。

    Note:
    一部のキーボードでは、CTRL-_ は CTRL-? にマッピングされることに注意。

  • ’hkmap’ が有效になつてゐる閒のキーボードマッピングは、次のやうになります (イスラエル標準キーボード): 
    q w e r t y u i o p
/ ' ק ר א ט ו ן ם פ

a s d f g h j k l ; '
ש ד ג כ ע י ח ל ך ף ,

z x c v b n m , . /
ז ס ב ה נ מ צ ת ץ .

    これはまた ’keymap=hebrew’ を設定したときのキーマッピングでもあります。’keymap’ の利點は、UTF8 を使用してゐるときに適切に動作することです。例へば、正しい文字を插入します。’hkmap’ はさうではありません。’keymap’ のキーボードはまた、niqud (撥音記號) や te‘amim (朗唱マーク) を插入することができます。 これらのマッピングがどうなつてゐるのかを調べるには、キーマップファイルの ’hebrew.vim’ などを見ませう。

バックワードの入力

もしも ’revins’ (reverse insert) オプションが設定されてゐるなら、插入することが後方に向けて生じます。これはヘブライ語を入力するのに使用することができます。

文字を入力した時にカーソルは移動せずにテキストが右方向に移動していきます。<BS> はカーソル下の文字を削除します。CTRL-W と CTRL-U はまた反對の方向に動作します。

<BS>, CTRL-W と CTRL-U は、’backspace’ オプションの設定がどのやうなものであつても、插入の開始や行末では停止しません。

逆の (rightleft モードでの？) 置換モードは (まだ) ありません。

もしも ’showmode’ オプションが設定されてゐるなら、逆插入モードが有效の時にステータス行に "– REVERSE INSERT –" が表示されるでせう。

’allowrevins’ オプションが設定されてゐるときに、逆插入モードは CTRL-_ 經由でも入ることができます。このモードは特別な機能を持つてゐます:

1 つ目に、キーボードマッピングはウィンドウの方向によつて變更されます。

  • left-to-right ウィンドウの時は、’revins’ がヘブライ語の文章の入力に使はれます。そのため、キーボードはヘブライ語 (’hkmap’ が設定) に變はります。
  • right-to-left ウィンドウの時は、’revins’ が英語の文章の入力に使はれます。そのため、キーボードは英語 (’hkmap’ がリセット) に變はります。

2 つ目に、CTRL-_ 經由で ’revins’ から出た時に、カーソルは (もしも可能なら) 入力されたテキストの最後に移動します。

rightleft ウィンドウ中でのペースト

rightleft ウィンドウ內でマウスによつてテキストをカット＆ペーストするときは、テキストは反轉されるでせう。なぜなら、文字は右から左へのファイルに插入されると同時に、左から右へのカットバッファから得るからです。これを囘避するには、ペーストをする前に ’revins’ を (CTRL-? や CTRL-_ で) トグルします。

ヘブライ文字と ’isprint’ 變數

ヘブライ語文字のコードには、’isprint’ 變數によつて定義された non-printable 範圍に存在してゐるものがあります。例へば Linux コンソールでは、ヘブライ語フォントのエンコーディングは 128 からはじまります。既定の ’isprint’ 變數は @,161-255 です。この結果、すべてのヘブライ語の文字は ‘~x’ のやうに表示されます。この問題を解決するには、set isprint=1,28-255 のやうにします。
